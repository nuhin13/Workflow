# docs/ — the guides

**Why this exists.** Different readers need different depths.

| File | Reader | Depth |
|---|---|---|
| `../README.md` | everyone | the manual with diagrams (start here) |
| `harness-guide.md` | builders of the harness | deep dive: layers, PM model, review chain, limitations |
| `HUMAN-GUIDE.md` | the human operator | your gates + 3 daily commands, in 5 minutes |
| `business/` | analyst + human | the BRD home (business source of truth) |
| `design/` | designer + frontend | design canon — Figma URL when linked |

**What it does NOT cover.** Product docs of a specific project — those
grow in `project/` through the pipeline.
